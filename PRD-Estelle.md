# PRD — 에스텔 (Estelle)

> 앱인토스 입점용 전체이용가 **회귀 미연시** 캐릭터 게임.
> 작성일: 2026-07-01 (스토리 v2 재설계: 2026-07-02)

## 1. 개요

공작영애 '에스텔'의 곁을 지키다 그녀의 파멸을 막지 못한 당신이, **회귀**하여 매일
그녀를 찾아가 인연(호감도)을 쌓으며 정해진 운명을 바꿔가는 회귀 미연시. 광고로 모은
코인으로 의상·악세서리(코스메틱)를 갈아입힌다. 상세 서사: `STORY-Estelle.md`.

- **플랫폼**: 앱인토스 (Apps in Toss) — WebView 웹앱, ait CLI(.ait) 패키징
- **등급**: 전체이용가 (스토어 자동심의 대체 목표)
- **장르/톤**: 회귀 미연시. **성별 무관 몰입**(주인공=2인칭 '당신', 성별·외모 비묘사),
  유대·헌신 중심(로맨스로도 읽힘). 설렘은 있되 선정성 배제 → 심사 안전선 엄수
- **수익구조**: 앱인토스 통합 리워드 광고 → 인게임 코인 → 코스메틱 구매
- **핵심 발상**: 게임 루프(매일 대화·선물) = 회귀자의 개입 = 그녀의 운명을 바꾸는 힘

## 2. 핵심 정책 제약 (조사 결과 반영)

| 항목 | 제약 |
|------|------|
| 광고 SDK | 앱인토스 통합 SDK만 허용 (AdMob 직접 연동 불가) |
| 리워드 광고 | 허용. 단 "클릭 시 보상" 문구 금지 → "광고 시청 후 보상" 포맷만 |
| 확률보상 | 코스메틱 직구매(가챠 아님)라 사행성 쟁점 없음 |
| 콘텐츠 | 선정성/성적 대상화 금지 → 전체이용가 톤 엄수 |
| 수익화 전제 | 사업자+게임제작업 등록+게임물 등급분류증명 필요 |

## 2.5 리텐션·광고 설계 (2026-07-03 개편 — 웹툰 문법)

- **기다리면 무료**: 시즌1 12화 순차 해금. 클리어 후 다음 화 24h 타이머(무료) 또는
  리워드 광고 즉시 해금. 돌아올 이유 = 보상이 아니라 다음 이야기 (포인트앱 문법 탈피)
- **오늘의 일상**: 일일 1회 미니 씬(14종 로테이션) → 호감도. 구 '대화하기 5회' 대체
- **수집**: 화별 명대사 카드 12종 갤러리 + 7일 연속 방문 한정 악세서리
- **광고 3스팟** (통합 SDK, "시청 후 지급" 문구만): ①다음 화 즉시 해금(리워드)
  ②코인 부족 시 충전(리워드, 원래 시도 자동 재개) ③에피소드 완료 후 전면
  — 단 자동재생되는 1화 뒤는 스킵(인트로 정책), 일일 씬 뒤 미노출
- QA 회귀: `node qa-test.mjs`, `node qa-interstitial.mjs` (dev 서버 5180 필요, `?epwait=초`)

## 3. 게임 루프

1. 접속 → 에스텔이 인사 (인연 단계/시간대 반영 대사)
2. **대화하기**(무료, 일일 횟수 제한) → 호감도 +
3. **광고 보고 코인 받기**(리워드 광고) → 코인 +
4. **일일 출석** → 코인 +
5. **옷장**에서 코인으로 코스메틱 구매 → 착용(무료 교체)
6. 호감도 티어 상승 → 새 대사/옷장 슬롯 해금

## 5. 코스메틱 (슬롯 기반, 가챠 아님)

- **의상(outfit)**: 기본/캐주얼/원피스/파티 — 색상 레이어로 표현
- **악세서리(accessory)**: 없음/리본/안경/모자 — 이모지 레이어
- 구매=코인 소모, 소유 후 교체는 무료

## 6. 기술 스택

- Vite + TypeScript, DOM 기반 렌더 (프레임워크 無 → WebView 경량)
- 상태 저장: localStorage
- 광고: `src/game/ads.ts` 어댑터 — mock ↔ 앱인토스 SDK seam
- 프로토타입 아트: CSS/이모지 플레이스홀더 (추후 실제 일러 교체)
